# C# 代码风格规范

## 1. 命名规则
- 类、接口、方法、属性：PascalCase
- 私有字段：下划线开头 _xxx
- 局部变量、参数：小驼峰
- 常量：全大写 + 下划线

## 2. 代码格式
- 缩进：4 空格
- 大括号：独占一行
- 逻辑块之间空一行，提升可读性

## 3. 注释要求
- 公共类、公共方法必须写摘要注释
- 复杂业务逻辑、算法步骤必须加说明

## 4. 禁止写法
- 魔法字符串、魔法数字
- 过多嵌套 if/for
- 全局静态滥用
- 业务逻辑写在 xaml.cs

## 5. MVVM 编码约束
- ViewModel 只做逻辑，不操作控件
- 所有界面变更靠属性绑定驱动
