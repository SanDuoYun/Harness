# 架构设计文档 · WPF

## 1. 整体架构
采用标准 MVVM 分层架构：
- View：XAML 界面、纯UI展示
- ViewModel：绑定、命令、业务逻辑、属性通知
- Model：数据实体、实体映射
- Service：数据访问、数据库、网络请求、第三方接口
- Common：全局工具、扩展方法、帮助类

## 2. 核心模块划分
- 登录认证模块
- 主窗口 & 导航模块
- 业务功能模块
- 数据读写模块
- 系统配置模块
- 全局日志 & 异常处理模块

## 3. 数据流向
1. 界面触发 Command 命令
2. ViewModel 接收并处理业务逻辑
3. 调用 Service 获取/保存数据
4. 绑定属性自动刷新 UI

## 4. 技术选型原则
- 轻量、稳定、适配桌面端
- 组件复用、低耦合、易扩展
- 统一版本、统一依赖管理
